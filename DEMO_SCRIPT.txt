[0:00 - HOOK]
Choosing what to wear each day that matches the weather and events while keeping track of your inventory can be exhausting. That's why I created the Wardrobe Stylist, a personalized fashion coordinator designed to simplify outfit selection and catalog management.

[0:20 - WHAT IT IS]
The Wardrobe Stylist is a secure, personal concierge agent built with the Google Agent Development Kit 2.0. It acts as an automated wardrobe manager and style advisor that integrates with local database tools and weather APIs.

[1:00 - ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through the architecture diagram to see how requests flow.
First, user queries enter the security_checkpoint node where sensitive PII is scrubbed and prompt injections are instantly caught.
Next, valid requests route to the wardrobe_orchestrator agent, which coordinates traffic and delegates tasks.
If the query is about cataloging or inventory, it is delegated to the wardrobe_manager sub-agent.
If it is a styling request, it goes to the stylist_advisor sub-agent.
These sub-agents communicate directly with the local database and weather systems using Model Context Protocol tools.
Finally, styling suggestions route to the human_confirmation node, where the workflow pauses using a RequestInput card, allowing the user to approve or reject recommendations before they are outputted.

[3:00 - SECURITY & MCP]
Our security checkpoint keeps data private with PII regex scrubbing and detailed JSON audit logs. Our MCP server exposes four distinct stdio tools that allow the LLMs to read and write wardrobe logs locally.
